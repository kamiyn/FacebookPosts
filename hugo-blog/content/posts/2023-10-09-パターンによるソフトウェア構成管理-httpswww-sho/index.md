パターンによるソフトウェア構成管理
https://www.shoeisha.co.jp/book/detail/9784798112596

原著は 2002年 Git がまだ無かった時の本
GitHub または GitLab を基盤とした CI/CD 環境では
当たり前のように実践されるような話が主

Smoke Test と Regsession Test の違いを認識できるのが
一番大きかった

具体的なパターンに入る前の解説では
ワインバーグ本からの引用が多数あるのが嬉しい

「計画立案の時に蚊帳の外に置かれた人々は、プロジェクト変更のときにきまって現れ、あなたを悩ませる」
は直前読んだ プロダクトマネージャーのしごと に通じる。

ソフトウェア構成管理もまたコミュニケーションの一部

「パターン」という言葉の説明では
クリストファー・アレグザンダー の  パタン・ランゲージ にある一節にも言及されていた。ITのアーキテクチャ分野から 建築としてのアーキテクチャは参照するものらしい。
